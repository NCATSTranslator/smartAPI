FAQ

SmartAPI

  • What is SmartAPI?
    The SmartAPI project aims to maximize the FAIRness (Findability, Accessibility, Interoperability, and Reusability) of web-based Application Programming Interfaces (APIs). Rich metadata is essential to properly describe your API so that it becomes discoverable, connected, and reusable. We have developed a openAPI-based specification for defining the key API metadata elements and value sets. SmartAPI's leverage the Open API specification v3 and JSON-LD for providing semantically annotated JSON content that can be treated as Linked Data.
  • Is there a guide to help me get started?
    If you are new to SmartAPI you can use our guide to help you contribute for the first time. The guide features examples, and easy step-by-step instructions on how to get started.
  • What's a dashboard? And how do I access it?
    Your user dashboard is place where you can easily manage and quickly access your registered APIs. To access it log in and click on your user image on the menu bar.

API Management

  • What is a slug and how do I register a custom slug for my API?

    A slug is a part of a URL which identifies a particular page on a website in a form readable by users. By default SmartAPI uses an API's ID as the slug to serve an API's documentation. For example you can view your API's documentation by visiting <id>.smartapi.info. However you can register for a more readable slug so you can share your documentation under a namespace that represents your API.

    To register a custom slug you must successfully add a new API to the registry, log in to your dashboard to see a list of your registered APIs. Click on the name of any API and go to the Settings tab. Use the Custom Slug Registration Wizard and get a custom URL for your API's documentation. E.g. <slug>.smart-api.info

  • How often is API metadata updated automatically?

    Our system performs checks for changes in your metadata daily and unless any are found that metadata will not be updated.

  • How do I manually refresh my API metadata if I just made a change?

    Log in and head over to your dashboard, click on the refresh button after making your changes to your file externally and our system will refresh your metadata.
    There may be a small delay for changes to be be detected from GitHub repositories.

  • How can I transfer/claim the ownership of an API entry?

    Transfer of ownership requests can only come from a registered user, and should be accompanied with a message detailing the rationale for the transfer. Some evidence should be presented that supports your/their claim about ownership (name listed on official website or in API documentation), along with other information to follow up (e.g. full name, email address, phone number, github account). Once reviewed, the current API owner can then transfer the ownership to the other user.

    If you would like to transfer ownership to someone else or claim ownership of an API please contact us.

  • My API uses Swagger V2 specification, does SmartAPI support it?

    You can submit Swagger V2 APIs and they will appear on our registry, however they will not experiece full functionality on SmartAPI and BioThings Explorer.

    If you use our guide you can upgrade your Swagger V2 data to OpenAPI V3 Specification

  • How do I delete an API?
    To delete an API you must be the registered owner of that API. Log in to access the user dashboard and see a list of the APIs you have registered. Click on the delete button of the API you want to delete and follow the instructions to delete.

Other

  • Can I use SmartAPI's branding on my project?

    For most cases yes as long as proper credit is given to our services. For more information please read our guidelines on our branding page.

  • I need additional help, how do I contact SmartAPI?

    If you need to contact us you can email us or if you spot a bug or have any suggestions you may open an issue on GitHub.
